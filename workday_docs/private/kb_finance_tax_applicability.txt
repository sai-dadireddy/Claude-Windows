OVERVIEW
================================================================================

Tax Applicability in Workday allows you to assign default tax codes to sales,
purchase, and expense items to aid transaction entry and approval. You can:
- Enable tax applicability for spend transactions, revenue transactions, or both
- Configure tax rules that specify override tax applicability
- Auto-populate tax applicability on transaction lines for specific items

================================================================================
FILTER OPTIONS
================================================================================

At transaction entry, you can filter tax applicability results by:
- Spend
- Revenue
- Inactive status
- Country

================================================================================
SUPPORTED TRANSACTION TYPES
================================================================================

Tax Applicability can be specified on these transaction line types:

SPEND TRANSACTIONS:
- Purchase Orders
- P-Card Transactions
- Requisitions
- Supplier Contracts
- Supplier Invoices
- Supplier Invoice Adjustments
- Supplier Invoice Requests
- Expense Reports

REVENUE TRANSACTIONS:
- Customer Invoices
- Customer Invoice Adjustments
- Cash Sales
- Down Payment Invoices

FINANCIAL TRANSACTIONS:
- Accounting Journals
- Ad Hoc Bank Transactions
- Ad Hoc Payments
- Intercompany Contracts

================================================================================
WORKDAY TASK: MAINTAIN TAX APPLICABILITY
================================================================================

Task: Maintain Tax Applicability
Navigation: Search > "Maintain Tax Applicability"

Purpose: Create or edit tax applicability configurations

Fields:
- Tax Applicability Name: Unique identifier
- Tax Applicability by Company's Country: Filter by company country
- Tax Applicability by Country: Filter by transaction country
- Spend/Revenue indicator: Which transaction types use this applicability
- Default Tax Code: Tax code applied when this applicability is selected
- Inactive: Mark as inactive to prevent selection

ELECTRON AUTOMATION STEPS
================================================================================

--- View Tax Applicabilities ---

1. enter search box as "All Tax Applicabilities"
2. wait for search results
3. click search result containing "All Tax Applicabilities"

5. verify list of tax applicabilities displays

7. enter search box as "Create Tax Applicability"

9. click search result containing "Create Tax Applicability"
10. wait for page to load
11. enter Tax Applicability Name as "{applicability_name}"
12. select Tax Applicability Type (Spend/Revenue/Both)
13. select Default Tax Code as "{tax_code}"
14. select Country if filtering by country
15. click button "OK"
16. wait for confirmation
17. verify message contains "created"
18. screenshot as "tax_applicability_created.png"

--- Edit Tax Applicability ---

19. enter search box as "Maintain Tax Applicability"
20. wait for search results
21. click search result containing "Maintain Tax Applicability"
22. wait for page to load
23. search for Tax Applicability to edit
24. click matching result
25. modify fields as needed
26. click button "OK"
27. wait for confirmation
28. screenshot as "tax_applicability_updated.png"

================================================================================
SOAP API OPERATIONS
================================================================================

WSDL: Revenue_Management

Operations:
- Get_Tax_Applicabilities: Retrieve tax applicability configurations
- Put_Tax_Applicability: Create/update tax applicability

Request Parameters:
- Tax_Applicability_Reference: ID of existing applicability (for update)
- Tax_Applicability_Data:
  - Tax_Applicability_Name
  - Tax_Category_Reference
  - Default_Tax_Code_Reference
  - Applicable_for_Spend (boolean)
  - Applicable_for_Revenue (boolean)
  - Country_Reference (optional)
  - Inactive (boolean)

================================================================================
TAX APPLICABILITY ON TRANSACTIONS
================================================================================

SUPPLIER INVOICE:
- Tax Applicability field on invoice line
- Determines withholding tax calculation
- Can override default from supplier/item

EXPENSE REPORT:
- Tax Applicability field on expense line
- Can be defaulted from expense item configuration
- Filters available tax codes for selection

PURCHASE ORDER:
- Tax Applicability on PO line
- Flows through to invoice matching
- Can trigger withholding tax calculation

REQUISITION:
- Tax Applicability on requisition line
- Non-catalog lines can have tax applicability
- Flows to PO when converted

================================================================================
CONFIGURATION BEST PRACTICES
================================================================================

1. Define tax applicabilities for each tax jurisdiction
2. Set up defaults on items to reduce manual entry
3. Use country filtering to simplify user selection
4. Configure separate applicabilities for Spend vs Revenue
5. Review inactive applicabilities periodically
6. Document tax applicability-to-code mappings
7. Test tax calculation with various applicabilities
8. Train users on when to override defaults

================================================================================

- Inactive Tax Applicability Report: Find inactive configurations

================================================================================
NOTES
================================================================================

- Tax applicability determines which tax codes are available
- Withholding taxes can be calculated using tax applicability
- Tax rules can override tax applicability on transaction lines
- Country-specific tax applicabilities improve user experience
- Inactive applicabilities cannot be selected on new transactions
- Changes to tax applicability don't affect posted transactions
