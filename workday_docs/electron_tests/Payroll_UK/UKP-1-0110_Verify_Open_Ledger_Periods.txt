DESCRIPTION:
Verify that there are open ledger periods in place for UK payroll. Open ledger periods control when journal entries can be posted to specific accounting periods. For UK Payroll, payroll GL entries post to open periods only and the period must be open for payroll settlement.

WORKDAY ROLE: Payroll Administrator
EST. DURATION: 5 minutes
COUNTRY SCOPE: UK

PREREQUISITES:
- User logged in with Payroll Administrator permissions
- Required data available in tenant
- Access to UK payroll data
- Knowledge of current fiscal year and payroll period

ELECTRON STEPS:
1. enter search box as "View Ledger Period Status"
2. wait for search results
3. click search result containing "View Ledger Period Status"
4. wait for page to load
5. select ledger as "Actuals"
6. select fiscal year as "{current_year}"
7. wait for periods to load
8. verify current month status is "Open"
9. verify prior month status (Open or Closed)
10. screenshot as "UKP-1-0110_ledger_periods.png"
11. note the open ledger periods
12. verify payroll payment dates fall within open periods
13. screenshot as "UKP-1-0110_complete.png"

VERIFICATION:
- [ ] Current payroll period is open
- [ ] Prior period open (if retro adjustments needed)
- [ ] Future periods not prematurely opened
- [ ] Adjustment period closed (except year-end)
- [ ] Actuals ledger period status correct
- [ ] No error messages displayed
- [ ] Country-specific data validates correctly

API ALTERNATIVE:
- SOAP: Financial_Management.wsdl - Get_Ledger_Periods, Get_Ledger_Period_Status
- REST: financialManagement API - /ledgerPeriods endpoint

NOTES:
- Period Status Values: Open, Closed, Future, Adjustment
- Ledger Types: Actuals, Budget, Commitment
- Fiscal year alignment required for payroll settlement
- Month-end closing requires period closure
