FUNCTIONAL AREA: Payroll for the United States
TEST NAME: Missing Payment Elections Review
CONFIDENCE: [HIGH] Score: 8.5/10
SOURCE: [KB Article: kb_payroll_missing_payment_elections.txt]
================================================================================

DESCRIPTION:
Ensure that all active employees in US companies have the applicable Payment
Elections loaded (incl. for any additional run categories). Note: The report
displays Missing elections, not a list of all existing elections.

WORKDAY ROLE: Payroll Administrator
EST. DURATION: 10 minutes

PREREQUISITES:
- User logged in with Payroll Administrator permissions
- Access to US company payroll data
- Companies configured with payment election rules

ELECTRON STEPS:
1. enter search box as "Audit Missing Payment Elections"
2. wait for search results
3. click search result containing "Audit Missing Payment Elections"
4. wait for page to load
5. verify Company field is displayed
6. select Company dropdown as "{US_company_name}"
7. wait for Company to resolve
8. select Run Category dropdown as "Regular"
9. click button "OK"
10. wait for report results to load
11. wait 5 seconds
12. verify results table displays or "No results found" message
13. scroll to view all results if present
14. verify columns include: Worker, Company, Pay Group, Missing Election Type
15. screenshot as "USP-1-0140_missing_elections_regular.png"
16. note count of workers with missing elections (if any)
17. click button "Close" or navigate back
18. enter search box as "Audit Missing Payment Elections"
19. wait for search results
20. click search result containing "Audit Missing Payment Elections"
21. wait for page to load
22. select Company dropdown as "{US_company_name}"
23. wait for Company to resolve
24. select Run Category dropdown as "Additional"
25. click button "OK"
26. wait for report results to load
27. verify results for Additional run category display
28. screenshot as "USP-1-0140_missing_elections_additional.png"
29. verify all active employees have applicable payment elections loaded
30. screenshot as "USP-1-0140_complete.png"

VERIFICATION:
- [ ] Report loads successfully for US companies
- [ ] Regular Run Category shows no missing elections (or lists those missing)
- [ ] Additional Run Category shows no missing elections (or lists those missing)
- [ ] All active employees have payment elections configured
- [ ] No error messages displayed
- [ ] Report columns display correctly (Worker, Company, Pay Group, Missing Election Type)

API ALTERNATIVE:
SOAP:
  Service: Payroll
  Operation: Get_Payment_Election_Rules
  Description: Retrieves Payment Election Rules from the Workday system
  Note: Direct audit requires custom integration combining worker data with election data

REST:
  N/A - Payment election audit not available via REST API

ADDITIONAL NOTES:
- If missing elections are found, use "Maintain Payment Elections for Worker" to add them
- Ensure bank accounts are configured before adding Direct Deposit elections
- Check both Regular and Additional run categories before each payroll cycle
- Payment Election Enrollment Event can trigger during hire process
