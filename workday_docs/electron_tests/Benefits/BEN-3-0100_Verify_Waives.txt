DESCRIPTION:
Verify benefit waivers to confirm employees who have declined coverage are
properly documented with valid waiver reasons. Extract worker benefit elections
details to identify waived benefits and ensure compliance with waiver policies.

WORKDAY ROLE: Benefits Administrator

PREREQUISITES:
- User logged in with Benefits Administrator permissions
- Active benefit enrollment period
- Employees who have waived benefit coverage

ELECTRON STEPS:
# Access Worker Benefit Elections Report
1. enter search box as "Extract Worker Benefit Elections Details"
2. wait for search results
3. click search result containing "Worker Benefit Elections"
4. wait for page to load
5. screenshot as "BEN-3-0100_report_access.png"

# Configure Report for Waivers
6. select prompt "Election Status" as "Waived"
7. select prompt "As Of Date" as "today"
8. select prompt "Benefit Plan Type" as "All"
9. click button "OK"
10. wait for report to generate
11. screenshot as "BEN-3-0100_waiver_filter.png"

# Review Waiver Details
12. verify report displays waived elections:
    - Worker Name
    - Waived Benefit Plan
    - Waiver Reason
    - Waiver Effective Date
    - Alternative Coverage Source
13. screenshot as "BEN-3-0100_waiver_list.png"

# Verify Waiver Reasons
14. verify valid waiver reasons documented:
    - Covered under spouse's plan
    - Medicare/Medicaid coverage
    - VA benefits
    - Retiree coverage
    - Cannot afford (if allowed)
15. screenshot as "BEN-3-0100_waiver_reasons.png"

# Confirm Waiver Compliance
16. verify waiver attestations completed:
    - Other coverage source certified
    - Re-enrollment restrictions acknowledged
    - Waiver effective date appropriate
17. verify no incomplete waivers exist
18. screenshot as "BEN-3-0100_complete.png"

VERIFICATION:
- [ ] Worker Benefit Elections report successfully generated
- [ ] Waived elections filter displays correctly
- [ ] All waived benefits show valid waiver reasons
- [ ] Waiver effective dates are appropriate
- [ ] Alternative coverage sources documented
- [ ] Attestations completed for all waivers
- [ ] No employees with incomplete waiver documentation
- [ ] Waiver counts match enrollment waiver totals
- [ ] Compliance with company waiver policies confirmed

EXPECTED RESULT:
Worker Benefit Elections report with waiver filter displays all employees
who have declined benefit coverage, with valid waiver reasons, alternative
coverage documentation, and appropriate attestation completion.

NOTES:
- Employees can waive coverage if covered under spouse or other source
- Valid waiver reasons: Spouse coverage, Medicare, VA, Retiree plan
- Waiving coverage restricts re-enrollment until next open enrollment
- Exceptions: Qualifying life events allow mid-year changes
- Loss of other coverage triggers special enrollment period
- Document waiver reasons for compliance audits
- Confirm waiver does not leave employee without required coverage

API ALTERNATIVE:
- SOAP: Benefits_Administration.wsdl
  - Get_Benefit_Elections (filter for waived status)
  - Change_Benefits (includes waiver processing)
- REST: /workers/{id}/benefits (shows election status)
- Report: Benefits Waiver Summary (standard Workday report)
