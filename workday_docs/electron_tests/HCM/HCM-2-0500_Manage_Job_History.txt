JOB HISTORY FIELDS:
- Job Title: Position held at previous employer
- Company/Organization: Previous employer name
- Industry: Business sector (from reference data)
- Start Date: When position started
- End Date: When position ended (leave blank if current)
- Location: Work location city/state/country
- Description: Responsibilities and achievements
- Contact Name: Reference contact (optional)
- Contact Information: Phone/email of reference (optional)
- Reason for Leaving: Why position ended (optional)

FIELD REFERENCE:
- Job Title: Free text field for position name
- Company: Free text or reference lookup
- Industry: Dropdown from configured industries
- Start Date: Date picker
- End Date: Date picker (optional)
- Location: Text field or location reference
- Description: Multi-line text field

ALTERNATIVE: Add from Worker Profile
1. Navigate to worker profile
2. click button "Related Actions"
3. click menu item "Career"
4. click menu item "Add Job History"
5. (continue from step 14 above)

ALTERNATIVE: Edit Existing Job History
1. Navigate to worker profile
2. click tab "Career"
3. scroll to Job History section
4. click existing job history entry
5. click button "Edit"
6. modify fields as needed
7. click button "Submit"

ALTERNATIVE: Administrator Entry
1. Login as HR Administrator (not proxied)
2. enter search box as "Add Job History"
3. Select worker when prompted
4. Enter job history details
5. Submit (may bypass approval)

API ALTERNATIVE:
- SOAP: Human_Resources.Put_Job_History
- REST: /workers/{ID}/jobHistory (POST)

ERROR HANDLING:
- "Worker not found": Verify worker name/ID
- "Invalid date range": End date must be after start date
- "Access denied": User lacks self-service access
- "Approval required": Job history awaiting manager approval
