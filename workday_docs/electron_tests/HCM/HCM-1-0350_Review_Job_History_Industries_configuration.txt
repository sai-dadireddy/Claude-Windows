DESCRIPTION:
Verify that all expected job history industries are present in the tenant. Job History Industries provides a list of industry classifications for use when recording a worker's previous employment history.

WORKDAY ROLE: HR Administrator
EST. DURATION: 5 minutes

PREREQUISITES:
- User logged in with HR Administrator permissions
- Access to HCM Reference Data domains
- List of expected industries for comparison (if available)

ELECTRON STEPS:
1. enter search box as "All Job History Industries"
2. wait for search results
3. click search result containing "All Job History Industries"
4. wait for page to load
5. screenshot as "HCM-1-0350_report_view.png"
6. verify report displays table with columns "Reference ID" and "Name"
7. scroll to view all industry entries
8. verify column headers include "ID", "Name", "Active"
9. click column header "Name" to sort alphabetically
10. wait 2 seconds
11. screenshot as "HCM-1-0350_sorted.png"
12. verify expected industries appear in the list
13. verify Active status shows "Yes" for active industries
14. click filter dropdown for "Active" column
15. select filter as "Yes" to show only active entries
16. wait for filter to apply
17. screenshot as "HCM-1-0350_active_filtered.png"
18. verify filter applied successfully
19. click link "Export to Excel" if export verification needed
20. screenshot as "HCM-1-0350_complete.png"

VERIFICATION:
- [ ] All Job History Industries report displays successfully
- [ ] Expected industries are present (Manufacturing, Healthcare, Finance, etc.)
- [ ] Active/inactive status correctly reflects configuration
- [ ] No duplicate industry entries exist
- [ ] Report can be filtered and sorted
- [ ] Task completed successfully
- [ ] No error messages displayed

DATA VALIDATION CHECKLIST:
- [ ] Standard industries present (Manufacturing, Healthcare, Finance, Retail, IT)
- [ ] Professional Services / Consulting industries present
- [ ] Education and Government sectors present if applicable
- [ ] Industry naming conventions are consistent
- [ ] Reference IDs follow expected pattern

API ALTERNATIVE:
- SOAP: No direct operation - Job History Industries is reference data
- SOAP: Related: Put_Previous_System_Job_History, Get_Previous_System_Job_History (Human_Resources.wsdl)
- REST: Reference Data API if configured for tenant

RELATED TASKS:
- Maintain Job History Industries: Add or edit industry entries
- Create Job History Industry: Add new industry to the list
- All Job History Companies: Review companies for job history
- Edit Prior Work Experience: Uses industries when entering worker job history

NOTES:
- Job History Industries populates industry dropdown in prior work experience entry
- Inactive industries will not appear in worker profile selection
- Consider alignment with NAICS or SIC codes for standard reporting
- This is reference data setup, not transactional data
