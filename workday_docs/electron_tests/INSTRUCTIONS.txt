================================================================================
HOW IT WORKS
================================================================================

For EACH scenario in the Excel file:

1. FILTER by Functional Area (Inventory or Asset Management)

2. EXTRACT data:
   - Scenario ID
   - Scenario Name
   - Task / Step
   - Customer Expected Result

3. QUERY RAG (if Task/Step exists):
   python workday_rag.py "{Task/Step}"

4. CALCULATE CONFIDENCE:
   - HIGH (8.0-10.0):   Detailed SOAP/REST operations, specific steps
   - MEDIUM (5.5-7.9):  General UI guidance, some context
   - LOW (0-5.4):       Minimal/no information

5. GENERATE ELECTRON STEPS:
   - Parse RAG response for actionable steps
   - Create step-by-step test instructions
   - Include verification against expected result

6. CREATE FILE:
   - Filename: {Scenario_ID}_{Scenario_Name}.txt
   - Location: Inventory/ or Asset_Management/

================================================================================
FEATURES
================================================================================

RAG CACHING
-----------
  - Duplicate Task/Step queries are cached
  - Significantly speeds up processing
  - Cache size reported in summary

RATE LIMITING
-------------
  - 0.5 second delay between RAG queries
  - Prevents overwhelming the RAG system
  - Ensures stable operation

ERROR HANDLING
--------------
  - Continues processing on individual failures
  - Logs errors for review
  - Creates LOW confidence files for failed queries

ISSUE: Permission errors creating files
