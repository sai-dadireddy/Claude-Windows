TASK/STEP: Create Case

DESCRIPTION:
Verify appropriate case types show up for various levels of Managers

PREREQUISITES:
- User logged in with Manager As Self role
- Case Management module enabled
- Service Categories and Case Types configured
- Case Eligibility Rules configured for Manager role
- User has direct reports (is a manager in Workday hierarchy)

ELECTRON STEPS:
1. enter search box as "Create Case"
2. press Enter or click Search
3. wait for search results to load
4. click search result containing "Create Case"
5. wait for page to load
6. screenshot as "HLP-4-0020-01_service_categories.png"
7. verify Service Categories are displayed (e.g., HR Services, IT Support, Payroll)
8. click tile for first Service Category
9. wait for Case Types to load
10. screenshot as "HLP-4-0020-01_case_types.png"
11. verify Case Types displayed include manager-specific case types
12. verify case types allowed by Case Eligibility Rules for Manager role appear
13. verify manager can see additional case types not available to Individual Contributors
14. verify case types related to direct reports are available (e.g., Team HR Issues)
15. click Back to return to Service Categories
16. repeat steps 8-12 for each Service Category to verify manager eligibility filtering
17. screenshot as "HLP-4-0020-01_complete.png"

VERIFICATION:
- [ ] Service Categories display correctly for Manager role
- [ ] Case Types include manager-specific options
- [ ] Manager sees more case types than Individual Contributor role
- [ ] Case Types related to direct reports are accessible
- [ ] Case Eligibility Rules properly filter available case types
- [ ] No error messages displayed

ALTERNATE NAVIGATION:
1. click worklet "Help"
2. click link "Create Case"
3. continue with step 6 above

API ALTERNATIVE:
- SOAP: Requests.Submit_Request - Creates a Request/Case
- Note: Case eligibility is UI-enforced, API does not filter by role
