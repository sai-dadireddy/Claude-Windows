DESCRIPTION:
Verify the case routes to correct Service Team when created by a Help Case
Team member with Case Creator role.

This test validates that when a Help Case Team member (Case Creator) creates
a case, the Case Routing Rules correctly assign it to the appropriate
Service Team. Help Case Team members typically have access to all case types
and can create cases on behalf of any worker.

PREREQUISITES:
- User logged in with Case Creator role (Help Case Team member)
- User is member of a Service Team
- All Case Types accessible to Case Creator
- Case Routing Rules configured
- Test data: Know expected Service Team for selected Case Type

NOTE: Case Creators typically have broader access than regular employees.

================================================================================
CASE CREATOR ROLE SPECIFICS:
================================================================================

Case Creators have special permissions:
- View all Service Categories and Case Types
- Create cases for any worker (including self)
- Access Help Dashboard for case management
- May be able to directly assign cases
- Not restricted by Case Eligibility Rules

This differs from employee/manager roles which see filtered case types.

================================================================================
HELP DASHBOARD ACCESS:
================================================================================

Case Creators access the Help Dashboard to:
1. View cases assigned to their Service Team
2. Manage case queues
3. Verify routing worked correctly
4. Work and resolve cases

The Help Dashboard shows:
- Case queue for assigned Service Team
- Case details including routing information
- SLA status and due dates
- Case history and activities

================================================================================
HOW TO IDENTIFY EXPECTED ROUTING:
================================================================================

Before running test, verify:

1. Search "Maintain Case Routing Rules"
2. Find rule for the Case Type being tested
3. Note the Service Team in "Route To" field
4. Verify Case Creator is member of that Service Team

================================================================================
ERROR SCENARIOS:
================================================================================

CASE NOT VISIBLE IN DASHBOARD:
- Verify Case Creator is member of the routed Service Team
- Check Service Team security access
- Verify case was actually created (check confirmation)

CASE NOT ROUTED:
- Verify Case Routing Rule exists
- Check routing rule conditions
- Review fallback "All Else Route To" team

WRONG SERVICE TEAM:
- Check routing rule priority order
- Verify routing conditions match case data
- Review multiple rules that may apply
