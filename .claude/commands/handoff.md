Write or update a **handoff document** so the next agent with fresh context can continue this work seamlessly.

## Process

1. **Check for existing HANDOFF.md** in the project root
2. **Review prior context** if the file exists
3. **Create/update** with the sections below

## Required Sections

```markdown
# Handoff Document

**Project:** [Project name]
**Date:** [Current date]
**Last Updated By:** Claude

---

## Goal
[What we're trying to accomplish - clear, concise statement]

## Completed Work
[What has been done so far - bullet points]
- [x] Task 1
- [x] Task 2
- [ ] Task 3 (in progress)

## What Worked
[Approaches and solutions that succeeded]
- [Successful approach 1]
- [Successful approach 2]

## What Didn't Work (Avoid These)
[Failed attempts so the next agent doesn't repeat them]
- [Failed approach 1 - why it failed]
- [Failed approach 2 - why it failed]

## Next Steps
[Concrete, actionable items to continue the work]
1. [Next step 1]
2. [Next step 2]
3. [Next step 3]

## Key Files
[Important files for context]
| File | Purpose |
|------|---------|
| `path/to/file` | [What it does] |

## Notes
[Any other relevant context, gotchas, or warnings]
```

## Output

Save as `HANDOFF.md` in the project root and confirm the file path.

---

Now analyze the current work and create/update the handoff document.
